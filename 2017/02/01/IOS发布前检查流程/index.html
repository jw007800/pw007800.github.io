流程定义

该流程遵循前提为：App已经通过QA测试并满足相关需求

定义iOS客户端发布时所遵循的流程

iOS开发工程师需要遵循该流程完成发布

其他角色了解该流程

角色及职责

描述该流程中所依赖的角色及其职责描述，同时明确责任

角色 职责

PO/PD 提供需求、上架信息、应用信息等

Wiki 记录上架信息及更新上架记录

All 相关干系人及部门，接收邮件通知

Server 服务器端，调整审核开关

Qiniu 保存上传二进制文件

输入项

该输入项默认为该版本的app所依赖的内容

CheckList 包含了app在提交时的检查清单列表

app相关代码 包含了需要提交的app的源代码

工具

描述了完成该流程所需要的工具

名称 描述

输出项

描述了经过该过程产出的内容列表

名称 描述

CheckList列表 记录app上架前的最后检测清单列表

流程概述图

描述发版流程走向

流程描述

正式版

发版详细描述该流程

PO/PD在Wiki iOS App上架记录 中更新上架应用相关信息

PO/PD发送邮件为iOS提供app上架相关信息内容

iOS根据邮件内容从Wiki中获取应用上架信息

iOS将应用上架信息上传iTunes，完成内容更新

iOS发送邮件将CheckList给QA，完成检测

iOS整理代码，执行clean -> compile -> archive，执行archive之后的二进制文件验证

iOS将archive文件上传iTunes，完成二进制文件上传

iOS在iTunes中选择该次编译版本，完成该版本的提交

iOS将上架记录及上架过程中出现的问题和解决方案更新到Wiki

iOS发送邮件通知PO/PD和All，完成该版本的发布

发版后审核通过描述流程

iOS或PO/PD收到iTunes中得反馈邮件

iOS更新Wiki文档，记录审核通过日期

iOS发送邮件通知PO/PD和All，告知该版本审核通过

PO/PD给iOS发送邮件为Release该版本

iOS通知Server调整审核开关，完成开关调整

iOS在iTunes中发布该版本

iOS发送邮件通知PO/PD和All，告知该版本已上线

发版后审核未通过描述流程

iOS或PO/PD收到iTunes中得被拒邮件

iOS将被拒原因更新到WIki文档

iOS发送邮件通知PO/PD和All，告知该版本被拒

需要申诉

PO/PD想iTunes提交申诉内容

功能调整

PO/PD给IOS发送需要调整的内容

iOS执行相应功能调整

企业版

发版详细描述该流程

PO/PD在Wiki iOS App上架记录 中更新渠道相关信息

PO/PD发送邮件为iOS提供app渠道信息内容

iOS根据邮件内容从Wiki中获取渠道信息

iOS发送邮件将CheckList给QA，完成检测

iOS整理代码，执行clean -> compile -> archive -> 导出spa

iOS将ipa文件上传iQiniu，完成二进制文件上传

iOS将上架记录及上架过程中出现的问题和解决方案更新到Wiki

iOS发送邮件通知PO/PD，完成该版本的发布
