iTerm - 让你的 MAC 命令行更加丰富多彩

今天要给大家介绍一个工具 iTerm

概述

iTerm 是一款完全免费的，专为 Mac OS 用户打造的命令行应用。我们可以打开它的官方网站看一看 www.iterm2.com

配色

Perferences -> Profiles 选项卡里面可以找到 Colors 配色方案面板在右下角的下来选框中，我们可以看到有几种内置的配色方案供我们选择，比如黑色风格，白色风格，深蓝风格，等等：

一款小工具 Go2Shell

自行下载，下载后拖到工具栏就可以,这样你想进入到哪个目录就不需要 cd 路径了，只需要在当前要进入的路径点击工具栏上的Go2Shell图标就可以了

一款小工具 autojump插件

brew install autojump 根据完装完的提示把命令添加到~/.zshrc中保存退出即可
autojump：zsh 和 autojump 的组合形成了 zsh 下最强悍的插件

j + ‘目录名字’

快捷键

搜索

Cmd + F

例如：ping 192.168.254.254 这是你cmd + F，在搜索框输入192 + Tab 会自动补全，你可以copy等等…任意操作。

显示和隐藏

cmd + 空格 当然你可以自己设置，Perferences -> Keys 左下角 Hotkey 选中复选框，自定义你的快捷键

标记某一行命令

这个功能简单来说就是，我们在某一行按下 Cmd + Shift + M 将这行标记下来。随后我们在需要的时候，可以按下 Cmd + Shift + J 就可以立即跳回这一行了。

剪贴板历史纪录

Cmd + Shift + H 这个也是一个很方便的特性,当你输入一个命令 copy 之后调用快捷键Cmd + Shift + H 你使用过的命令就会出现在剪贴板上，从此以后，尽情的复制和粘贴吧～

标签

新建标签 Cmd + t
关闭标签 Cmd + w
切换标签 Cmd + 数字 cmd + 左右 可切换标签
切换全屏 Cmd + t

分屏

垂直分屏 Cmd + d

水平分屏 Cmd + shift + d

切换屏幕 Cmd + option + 方向

查看历史命令 Cmd + 有问题

其它

清除当前行：ctrl + u

到行首：ctrl + a

到行尾：ctrl + e

前进后退：ctrl + f/b (相当于左右方向键)

上一条命令：ctrl + p

搜索命令历史：ctrl + r

删除当前光标的字符：ctrl + d

删除光标之前的字符：ctrl + h

删除光标之前的单词：ctrl + w

删除到文本末尾：ctrl + k

交换光标处文本：ctrl + t

清屏1：command + r

清屏2：ctrl + l

GO环境安装

发表于 2017-01-17 | 分类于 GO开发

安装

brew install GO

安装完成后可以命令 go version 查看版本

go env 查看环境

需要说明的是：

GOPATH:日常开发的根目录

GOBIN:是GOPATH下的bin目录

设置环境变量

vim .bash_profile

export GOROOT=/usr/local/Cellar/go
export GOPATH=/Users/songjie/Developer/GO
export GOBIN=$GOPATH/bin
export PATH=$PATH:$GOBIN

退出vim,使用source ~/.bash_profile即可完成对全局变量的配置了，可以在终端中输入go env查看配置后的效果:

DrawingBoard - oc

13年作过的东西，一直没有总结，这次单独抽出来和大家分享。

概述

这个画板本来分为老师端盒学生端，其实就是作了下适配，教师端多了一些其它功能，包括，录屏，airPlay投影、 远程协助、推送资料等。这里将和画板没有关系的内容去掉了，如果有朋友需要 airPlay投影appleTV、Scoket、远程协助的了解可以联系我。
code

需求

1 可以用手指画你想画的任何东西

2 用户可以改变线条的颜色粗细。颜色、透明度

3 保留每一种工具（笔、矩形、荧光笔…）的状态

4 可以分页

5 前进、后退、清空

6 添加备注、涂鸦。涂鸦可旋转缩放改变大小

框架设计

LZXDoodleViewController rootVC

browserTableView 分页控件

canvasBar 工具栏

ACEDrawingView 画布

如何表现涂鸦

1 屏幕上画什么？点、矩形、圆、直线、文字、图片

2 用组合结构来表示痕迹，通过组合模式可以把线条与点组合到树状结构，无论线条还是点，其实都是本质上的某种“痕迹”mark

组合结构的Mark模型的类图

父类型是一个协议，mark为所有具体类定义了熟悉和方法

3 绘制涂鸦

重载drawRect方法，touchesMoved 可以向UIView 发送 setNeedDisPlay消息，currentTool 子类（继承了mark协议的 矩形、线、圆）进行绘制

4 命令模式，在工具栏上的按钮（删除、撤销）都包含一个LZXBarCommand对象，每个对象都有一个execute() 方法
